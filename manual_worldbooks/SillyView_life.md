# SillyView_life 手动世界书

用途：存放与市场曲线无关的生活财务指令，例如开局资金、现实收入、生活支出、贷款、还款、旧债等。

建议世界书名称：`SillyView_life`

## 条目：SillyView_生活财务指令_总则

```text
这些指令用于处理现实生活、银行卡余额、债务和开局设定，不用于解释市场曲线或交易盈亏。

交易软件可以直接通过银行卡交易。不区分交易账户与银行卡账户的资金。

玩家的交易盈利和亏损由 SillyView 插件根据持仓和价格自动计算，不需要也不应该使用增加现金指令伪造交易盈利。

只有工资、奖励、支出、生活事件、贷款、旧债等现实财务变化，才使用本世界书中的指令。

所有指令都必须被包裹在一个位于消息末尾的单一 <command> 块中。

在其他地方提及指令时，不要完整输出可执行指令，避免被插件误解析。可以使用缩写，例如“可以使用 Player.AddCash 指令”。
```

## 条目：SillyView_现金变动指令

```text
[Player.AddCash(amount, "reason")]：增加现金。

只用于现实中的增加现金，例如工资、奖金、赞助、补贴、礼金、意外收入、新手启动资金等。

[Player.SubtractCash(amount, "reason")]：扣除现金。

用于生活支出、罚款、账单、维修、医疗、消费、被盗损失等。

不要用 Player.AddCash 表示投资盈利。

不要用 Player.SubtractCash 表示交易亏损。

交易相关盈亏由 SillyView 插件根据市场价格和持仓自动处理。
```

## 条目：SillyView_债务与贷款指令

```text
[Loan.Grant(amount, "reason")]：提供贷款，同时增加现金和债务。

适用于银行放款、临时借款、融资机会、剧情中获得一笔可用借款等。

[Loan.Repay(amount, "reason")]：要求还款，同时减少现金和债务。

适用于强制还款、主动还款、债务催收、银行扣款等。

[Player.AddDebt(amount, "reason")]：仅增加债务，不给予现金。

用于设定故事背景中的已有欠款、旧日赌债、历史贷款、欠款继承等。

如果贷款已经被花完，只需要使用 Player.AddDebt 增加债务，不要同时增加现金。
```

## 条目：SillyView_开局资金示例

```text
普通开局：

玩家获得一笔启动资金时，可以使用 Player.AddCash。

示例：
<command>[Player.AddCash(100000, "新手启动资金")]</command>

有负债开局：

银行账户里有五万块，但初始负债十万块。

示例：
<command>[Player.AddCash(50000, "新手启动资金")]</command>

示例：
<command>[Player.AddDebt(100000, "先前的债务，贷款已经被花完")]</command>
```

## 建议启用方式

```text
将以上四个条目添加到世界书 SillyView_life。

建议绑定为角色卡附加世界书，方便随角色卡导出。

这些条目可以常开，也可以按你的角色卡设计改成关键词触发。
```
