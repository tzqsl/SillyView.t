# SillyView.t

SillyView 是一个 SillyTavern / Tavern Helper 模拟市场插件。它会为当前角色创建独立世界书，保存账户、行情、新闻、AI 市场目标和交易记录。

## 主要功能

- 纯外汇行情：EURUSD、GBPUSD、USDJPY、AUDUSD、USDCAD、USDCHF。
- 图表视图：分钟、小时、日线均可切换蜡烛图或收盘价折线图，并保留成交量与持仓价格线。
- 交易系统：现货与杠杆独立持仓，支持做多、做空、加仓、平仓、手续费、点差和滑点；强平与资金费率仅作用于杠杆仓。
- 风控系统：强平、止盈、止损，持仓后可调整止盈止损。
- 资金系统：初始 10000 信用点、贷款、贷款利息、杠杆资金费率。
- 易上手交易：10% / 25% / 50% / 75% / 全仓快捷金额、做多保护、做空保护、实时风险档位。
- 多角色账户：初始化时扫描角色卡绑定世界书中的开户行格式，自动创建 `SillyView_accounts` 总控世界书，并把多个账号写入其中的独立状态词条。
- 市场推进：AI 导演模式和本地快速模式，快速模式支持 5/15/30 分钟、1小时和1天推进。
- 实时自动推进：可在初始化或设置中开启，每分钟推进一次分K，每60分钟执行一次后台整点结算；手动推进会重置计时。
- 上下文同步：生成 `sv_dialogue_context`；多角色账户与行情数据默认隐藏，由角色观察指令按需临时提供。
- 任务与绩效：新手任务板、当前净值、总收益率、最大回撤、已实现盈亏和胜率。

## 使用方式

1. 在 SillyTavern 中启用插件并打开面板。
2. 为当前角色创建 SillyView 世界书。
3. 在资产页查看任务板，按“下一步”提示完成基础目标。
4. 在交易页选择资产，用 10% / 25% / 50% / 75% / 全仓快捷金额填入交易金额。
5. 点击“做多保护”或“做空保护”自动填入止盈止损。
6. 使用“结束回合”让 AI 推进市场，或打开“快速模式”本地推进行情。
7. 在资产页查看账户、仓位、风控、任务进度和绩效。

## 新手流程

推荐第一局这样玩：

1. 选择 EURUSD 或 GBPUSD。
2. 用 10% 快捷金额开一笔小仓位。
3. 使用对应方向的保护预设。
4. 推进 1 小时或结束回合。
5. 观察未实现盈亏、新闻和任务板变化。
6. 盈亏达到目标后平仓，完成第一轮复盘。

## AI 指令

AI 市场导演应在一个 `<command>...</command>` 块内输出指令，例如：

```text
[Market.SetLongTarget("EURUSD", 1.1000, 12, "bull_trend", "美元走弱推动欧元上行", 0.7)]
[Market.Advance("EURUSD", "HOURLY", 1.0920, "bull_run")]
[Market.AddNews("EURUSD", "欧洲通胀数据高于预期，欧元短线获得支撑", 6)]
[Time.Set("2025年09月22日-星期一-10:00", "上午", "秋季", "晴")]
```

`Market.AddNews` 的第三个参数是新闻影响市场的小时数，范围为 1-168。新闻会永久进入前端归档，但只在有效期内发送给后台市场 AI。

每次后台 AI 回合都会同时处理全部可用货币：分别推进价格，并为每个货币维持长线和短线目标。新闻可随机归属于任意可用货币或 `GLOBAL`，不受当前查看货币限制，但必须符合外汇因果关系。

如果 AI 没有推进必要资产，插件会用本地市场模拟补齐，避免行情卡住。

## 多角色账户

初始化时，插件会扫描当前角色卡绑定的主世界书和附加世界书。只要某个条目里存在完整开户行格式，就会自动：

1. 创建或更新 `SillyView_accounts`，并绑定到角色卡附加世界书。
2. 按开户行账户数量在 `SillyView_accounts` 内写入多个 `sv_account_state_*` 独立账号状态词条。
3. 在 `SillyView_accounts` 写入始终启用的观察/交易指令和精简账户目录；实时账目、账户状态与唯一一份 `sv_kline_context` 默认关闭。

角色 AI 判断角色确实会查看手机或账户时，可以在首轮回复中输出：

```text
<command>
[Observe.Account("acct_x")]
[Observe.Account("acct_y")]
</command>
```

插件会合并同一轮的观察需求，临时启用一份 `sv_kline_context` 和对应的 `sv_account_state_*` 条目。账户状态包含最多 20 条近期重大事件；二轮决策完成后，临时条目会重新关闭，本次看到的事件会标记为已观察。设置页的“角色观察调试”窗口可在角色 AI 生成功能接入前模拟完整的解析、激活和清理过程。

## 角色决策管线

设置中开启“发送前生成心理与行为”后，每次用户发送消息都会在前台生成前执行一次静默角色决策：

1. 从此前最近一条助手消息提取 `<content>...</content>` 正文；存在多段时按顺序合并，没有标签时使用原正文。
2. 读取当前用户消息，并与上一条正文一起发送给独立的静默角色决策请求。
3. 角色可以输出心理、行为和 `Observe` 指令；观察最多连续进行四轮，每轮可同时查看任意多个角色账户。
4. 只有本轮实际观察过的账户才允许执行交易指令。
5. 命令块由插件处理并删除，剩余心理和行为结果通过一次性系统提示注入当前前台生成。

角色决策请求可以使用酒馆当前模型，也可以在设置页配置独立 API、密钥和模型。它不读取普通聊天历史或市场后台提示词，而是直接读取 `SillyView_accounts` 中的角色人设、`sv_accounts_trade_commands` 和 `sv_role_output_rules`。单独开启“自动记录截取调试”不会调用模型，只会在每次用户发送时记录消息编号、上一条正文和当前输入，并实时显示在角色观察调试窗口。

### 导入角色人设

在当前角色绑定的任意普通人设世界书条目中加入无参数初始化标记，并用角色名成对标签分隔内容：

```text
[SillyView.ImportRoleProfiles()]

<林浅>
林浅的人设内容……
</林浅>

<苏晚>
苏晚的人设内容……
</苏晚>
```

插件初始化时会把每个角色块分别复制为 `SillyView_accounts` 中关闭的 `sv_role_profile_*` 条目。每次初始化都会按源条目重新同步，已删除的人设不会残留。角色决策 AI 每轮都会读取并发送全部已导入人设，不做人名筛选或动态激活。

`SillyView_fx` 仍作为附加市场世界书上下文使用；其中旧版重复的 `sv_kline_context` 会被清理，最新摘要只保存在 `SillyView_accounts`。

推荐开户行文本格式：

```text
开户行: 星海商业银行
户名: 林浅
账号: 6222-0001
余额: 50000
负债: 10000
```

也支持 JSON：

```json
{
  "开户行": "星海商业银行",
  "户名": "林浅",
  "账号": "6222-0001",
  "余额": 50000,
  "负债": 10000
}
```

AI 从 `sv_accounts_trade_commands` 内的精简账户目录获取 account_id，观察对应账户后再使用：

```text
[Trade.Buy("account_id", "EURUSD", 1000, 5, 1.1000, 1.0600)]
[Trade.Sell("account_id", "GBPUSD", 500, 5, 1.2300, 1.3100)]
[Trade.SetRisk("account_id", "EURUSD", 1.1050, 1.0550)]
[Trade.CloseLong("account_id", "EURUSD")]
```

`Buy` / `Sell` 与 UI 语义一致：买入会开多/加多/平空，卖出会开空/加空/平多。更明确的指令包括 `OpenLong`、`OpenShort`、`AddLong`、`AddShort`、`CloseLong`、`CloseShort`。

## 数据说明

核心数据保存在当前角色绑定的世界书中：

- `sv_config`：插件配置和可用资产，内部条目，默认关闭。
- `sv_global_market`：时间和宏观状态，内部条目，默认关闭。
- `sv_player_portfolio`：现金、债务、持仓、交易记录、资产曲线，内部条目，默认关闭。
- `sv_asset_*`：每个资产的 K 线和当前价格，内部条目，默认关闭。
- `sv_dialogue_context`：给普通对话 AI 的市场摘要。
- `sv_kline_context`：紧凑 K 线摘要；只保存在 `SillyView_accounts`，默认关闭并由观察指令临时启用。
- `sv_market_overview`：后台市场 AI 专用的24小时总观摘要；只保存预计算结果和最近4个小时收盘价，不包含原始分钟K、账户或持仓数据。
- `sv_ai_context`、`sv_market_targets`：后台状态，默认关闭，不发送给角色 AI。
- `sv_news_archive`：永久新闻归档，仅供前端新闻板块读取，默认关闭。
- `sv_market_news_active`：仍在影响市场的限时新闻，默认关闭，由插件按需发送给后台市场 AI。
- `SillyView_accounts`：多角色协同交易总控附加世界书。`sv_accounts_trade_commands` 和 `sv_role_output_rules` 默认关闭，由角色决策请求按需直接读取；K 线摘要和 `sv_account_state_*` 由观察指令临时启用。`sv_accounts_index` 是账户读取和迁移所需的内部活动索引，始终关闭。
- `sv_auto_event_log`：保存在 `SillyView_accounts` 中的自动推进重要事件日志，记录强平、止盈止损、长线目标到期和整点结算时间；默认关闭，最多保留100条。
